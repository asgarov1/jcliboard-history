# ClipboardWatcher

A lightweight clipboard history manager for Linux, written in Java. Keeps the last 100 items you've copied, persists history across restarts, and shows a searchable popup on demand.

---

## Requirements

- Java 11 or newer (`java -version` to check)
- A GNOME or KDE desktop environment (for startup and shortcut steps)

---

## 1. Place the file

Copy `ClipboardWatcher.java` to a permanent location, for example:

```bash
mkdir -p ~/tools/clipboard
cp ClipboardWatcher.java ~/tools/clipboard/
```

> History is saved as `clipboard_history.txt` in the **same folder** as the `.java` file, so keep it somewhere stable.

---

## 2. Add aliases to `~/.bashrc`

Open `~/.bashrc` in any editor and add these lines at the bottom:

```bash
# Clipboard Watcher
export PATH_TO_CLIPBOARD_WATCHER="$HOME/tools/clipboard"

alias cw='javac $PATH_TO_CLIPBOARD_WATCHER/ClipboardWatcher.java && java -cp $PATH_TO_CLIPBOARD_WATCHER ClipboardWatcher'
alias ct='java -cp $PATH_TO_CLIPBOARD_WATCHER ClipboardWatcher trigger'
```

Then reload your shell:

```bash
source ~/.bashrc
```

**What the aliases do:**

| Alias | What it does |
|-------|-------------|
| `cw`  | Compiles the file (if changed) then starts the clipboard daemon in the background |
| `ct`  | Sends a toggle signal to the running daemon — opens or closes the history popup |

---

## 3. Start `cw` automatically on login

### GNOME

1. Open **GNOME Tweaks** → **Startup Applications** → click **+**
    - If you don't have GNOME Tweaks: `sudo dnf install gnome-tweaks`
2. Set the command to:
   ```
   bash -c "source $HOME/.bashrc && cw"
   ```
   Or use the full explicit command to avoid shell sourcing issues:
   ```
   bash -c "javac $HOME/tools/clipboard/ClipboardWatcher.java && java -cp $HOME/tools/clipboard ClipboardWatcher"
   ```

**Alternative (works without GNOME Tweaks) — autostart `.desktop` file:**

```bash
mkdir -p ~/.config/autostart
cat > ~/.config/autostart/clipboard-watcher.desktop << 'EOF'
[Desktop Entry]
Type=Application
Name=Clipboard Watcher
Exec=bash -c "javac $HOME/tools/clipboard/ClipboardWatcher.java && java -cp $HOME/tools/clipboard ClipboardWatcher"
Hidden=false
NoDisplay=false
X-GNOME-Autostart-enabled=true
EOF
```

This file will be picked up automatically on every login — no extra tools needed.

### KDE

1. Open **System Settings** → **Startup and Shutdown** → **Autostart** → **Add login script**
2. Enter:
   ```
   bash -c "javac $HOME/tools/clipboard/ClipboardWatcher.java && java -cp $HOME/tools/clipboard ClipboardWatcher"
   ```

---

## 4. Bind `ct` to a keyboard shortcut

### GNOME

1. Open **Settings** → **Keyboard** → scroll to the bottom → **Custom Shortcuts** → click **+**
2. Fill in:
    - **Name:** `Clipboard History`
    - **Command:** `bash -c "java -cp $HOME/tools/clipboard ClipboardWatcher trigger"`
    - **Shortcut:** press your preferred key combo (e.g. `Super + V`)

> **Note:** The command must be the full `java -cp ...` form, not the `ct` alias — GNOME shortcuts don't source `~/.bashrc`.

### KDE

1. Open **System Settings** → **Shortcuts** → **Custom Shortcuts** → **Edit** → **New** → **Global Shortcut** → **Command/URL**
2. Set:
    - **Trigger tab:** assign your key combo
    - **Action tab → Command:** `bash -c "java -cp $HOME/tools/clipboard ClipboardWatcher trigger"`

## 5. Usage

| Action | How |
|--------|-----|
| Start the daemon | Run `cw` in a terminal, or log out and back in if autostart is configured |
| Open/close popup | Press your shortcut key, or run `ct` in any terminal |
| Copy an item | Click any row in the popup — it copies and closes automatically |
| Search history | Start typing in the search bar at the top of the popup |
| Close popup | Press `Esc`, click outside, or click **Esc to close** in the header |
| Clear history | Right-click the system tray icon → **Clear History** |
| Stop the daemon | Right-click the system tray icon → **Exit** |


## Troubleshooting

**Popup doesn't open with the shortcut**
Make sure the daemon is running first (`cw`). The trigger command does nothing if the daemon isn't up.

**History not saving**
Check that the process has write permission to `~/tools/clipboard/`. You can verify with `ls -la ~/tools/clipboard/`.

**`cw` alias not found after editing `.bashrc`**
Run `source ~/.bashrc` in your current terminal. New terminals will pick it up automatically.

**Daemon already running error on second `cw` call**
The daemon is already up — just use `ct` to toggle the popup. To restart, kill the old process first: `pkill -f ClipboardWatcher`.
